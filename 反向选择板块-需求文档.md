# 反向选择板块 — 需求文档

> 整理自 git 提交记录，原始需求来自用户与 AI 的对话。当前（2026-07-23）已回退到稳定版本 e0497d4，反向选择板块已移除。以下为完整的功能设计记录，供后续重新实现时参考。

---

## 一、功能概述

在日记页新增「反向选择」模式，替换原来「自由书写」。核心概念来自谢小树课程：「旧程序说跑，你选择了站住」。

用户记录一次**矫正性情感体验**——当旧的行为模式触发时，你选择做了不一样的事，然后把这种「反向选择」作为身体证据记录下来。

---

## 二、UI 结构

### 第一版（57b917e）
- 单一表单页，一次性填写所有字段
- 字段：触发、旧程序会说、我选择了反向、结果（均为单行 input）
- 底部有情绪强度滑动条（触发时 / 现在），各 1-10
- 「保存并获取小树见证」按钮
- 保存后小树调用 AI 生成见证回应
- 列表展示，可展开查看详情、删除、导出
- 随机回顾卡片（reverse-sparkle）：从历史记录随机抽一条展示，可换一张

### 第二版 — 分步重构（5462f05）
- 改为 **4 步流程**：触发 → 旧程序 → 反向选择 → 结果
- 每步一个输入框，上一步/下一步导航
- 情绪强度分开放：触发时在第 1 步，现在在第 4 步
- 步骤指示器 + 提示文字
- 完成后显示汇总 + 小树见证 AI 回应
- 计数文案：「至春已经选择了 N 次反向」
- 副标题：「大脑听到的道理永远都无法成为你身体的部分。改变一定是在体验中发生。」

## 三、数据模型

```js
// 存储：this.freeDiaries (localStorage key: xs_free_diaries)
{
  id: Date.now(),
  trigger: "",           // 触发事件
  oldProgram: "",        // 旧程序会说
  newChoice: "",         // 我选择了反向
  result: "",            // 结果
  triggerIntensity: 5,   // 触发时情绪强度 1-10
  resultIntensity: 5,    // 现在情绪强度 1-10
  feedback: "",          // 小树见证 AI 回应
  createdAt: Date.now(),
}
```

## 五、随机回顾卡片（reverse-sparkle）

- 反向选择列表上方显示
- Fisher-Yates 洗牌循环，不重复
- 显示：日期、反向选择内容、触发/旧程序/结果/情绪详情
- 「换一张」按钮

## 七、哪些改动需要同步

添加反向选择板块时，以下地方需要联动修改：

1. **index.html** — `#free-diary` 区域的完整 HTML
2. **app.js** — 数据属性、保存/渲染/删除/导出方法、事件监听、周报格式
3. **style.css** — 反向选择相关全部样式
4. **app.js — 数据兼容**：`loadData()` 中已有旧 free 数据的迁移逻辑，无需修改

## 九、已知问题（需避免）

1. **所有 `document.getElementById` 必须做 null 检查**，否则某个 tab 内的元素在其他 tab 不存在时会崩溃
2. **随机回顾卡片**在记录为 0 条时不能渲染，否则 `detail.innerHTML` 报错
3. **4 步流程**中反向选择步骤使用了 `e.target.dataset.mode`，需确认 dataset 存在
4. **init() 中的 try/catch** 会吞掉错误，排查困难——不要过度包装
5. **Service Worker 缓存版本**修改后需同步更新 sw.js 和 index.html 中的版本号
6. 从「自由书写」改为「反向选择」后，`localStorage` 中旧 key `xs_free_diaries` 保持不变，数据迁移在 `loadData()` 中已有处理
